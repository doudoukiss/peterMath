# peterMath：Lenia 数学生命教学游戏

## 创意来源

我想把一个数学规则做成可以玩的教学游戏。评委打开后不需要先读说明，而是可以完成几个小任务：唤醒生命场、塑造生命、调规则、比较同一数据的两种显示方式，最后导出证据。

界面第一步会直接提示 `1 选工具 / 2 选任务 / 3 点生命场`。评委不是先读长说明，而是先点击、拖动、运行和导出，再看任务反馈和数学卡片。

## 数学规则

作品只保留一个核心系统：Lenia 连续生命场。

每个点有一个生命量 `u`。程序先计算周围邻居的加权平均 `K*u`，再用增长函数 `G` 判断这个点应该增长还是衰减，最后加入阻尼：

```text
u_next = clamp(u + dt*(G(K*u) - damping*u), 0, 1)
```

这里的意思是：合适的邻域会增长，太少或太多都会衰减。很多点一起这样变化，就会形成漂移、相互影响、开花、湍动和衰退。

## 工程实现

程序使用 Rust 开发，目标是生成 Windows 可运行程序。打开后默认进入中文任务模式。评委可以按任务玩，也可以切换到自动讲解或专家证据面板，调整预设、种子、画笔、盖章、卷积半径、增长中心、增长宽度和阻尼。

## 解释层

作品不只展示画面，还把数学原因放在任务后的“数学卡片”里：

- 数学原始图显示原始数值场。
- 艺术表达图用同一个数值场生成更有美感的颜色、轮廓和脊线。
- Field Inspector 显示某一点的数值、变化量、梯度、卷积结果、增长响应和预测下一步。
- Kernel Lens 显示邻域核半径和核曲线。
- 指标曲线显示 mass、entropy、stability、vitality 的变化。
- Active Region 显示活跃区域、中心点和漂移方向。
- 规则变量对照可以用同一个 Lenia 初始状态比较一个参数改变后的指标差异。

## 分享与证据

程序可以导出截图、参数 JSON、share-state JSON 和 evidence pack。导出的 JSON 会记录当前任务、种子、参数、指标和步数。这样评委可以看到同一个任务、同一个种子、同一组参数和同一帧画面之间的对应关系。

## 创新点

- 把一个连续场公式变成可玩的教学任务。
- 用清楚的 `1 选工具 / 2 选任务 / 3 点生命场` 路径帮助评委立刻上手。
- 用画布 coachmark 和操作反馈说明点击已经改变了当前 Lenia 场。
- 先让评委操作，再用检查器、指标和数学卡片解释美感从哪里来。
- 用任务反馈和参数对照帮助评委看见“涌现”不是随机动画。
- 同时提供原生程序和离线网页备用展示，降低评审现场运行风险。

## 实验结论示例

当我只改变 Lenia 的 growth center 或 kernel radius 时，图案会从稳定团块变成漂移、开花或衰减状态。说明复杂形态可能来自简单规则和参数之间的关系。
