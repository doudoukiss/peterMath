# peterMath Reference Projects

Use external repositories only as narrow references. The current competition product is a Lenia teaching game, so outside code should not change the repository root, add extra simulation worlds, or turn the app back into a technical demo.

## Primary Reference: Lenia

Lenia remains the conceptual foundation because it is continuous, rule-driven, visually organic, parameter-driven, and teachable through short missions.

Useful ideas to study:

- deterministic species/preset management;
- radial kernels and growth functions;
- compact explanations for local rule -> global behavior;
- high-quality continuous-field rendering;
- save/share state formats.

## UI References

Look for examples of short interactive learning loops:

- one visible objective;
- immediate feedback;
- optional hint;
- explanation unlocked after action;
- export/share as the final proof step.

## Rule

Do not import large architectures blindly. Preserve `peterMath` naming, Rust/egui delivery, Windows artifact flow, deterministic seeds, Lenia presets, mission mode, and evidence exports.
