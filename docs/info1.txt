peterMath current context
=========================

The project has converged to one competition identity: a Lenia mathematical life teaching game.

Goal
----
Create a playable teaching game that lets judges immediately interact with a living mathematical field. The game should teach that local field rules can generate coherent form, motion, instability, decay, and exportable evidence.

Core idea
---------
Each point in the Lenia field stores a scalar value `u`. On each step, the app computes a local weighted neighborhood `K*u`, maps it through a growth response `G`, subtracts damping, and clamps the result:

```text
u_next = clamp(u + dt*(G(K*u) - damping*u), 0, 1)
```

The artistic view must be derived from the same scalar field: value, gradient, contour bands, and frame delta. There should be no decorative particles and no unrelated visual systems.

Native direction
----------------
- Native Rust app is primary.
- GPU Lenia is the main viewing path.
- CPU Lenia remains fallback/reference for metrics, inspector, previews, and exports.
- Chinese UI is primary.
- The app opens into mission-based `任务模式`.
- First-run onboarding shows `1 选工具 / 2 选任务 / 3 点生命场`.
- Guided Lenia show mode remains a secondary presentation option.

Implemented pillars
-------------------
- Lenia presets: orbital field, twin organisms, coral drift/fading, kernel ring, sparse soup, dense bloom.
- Brush and stamp editing.
- Undo/redo.
- Raw and Artistic views from the same field.
- Inspector for value, previous value, delta, gradient, convolution, growth, and estimated next value.
- Metrics: mass, entropy, symmetry, stability, vitality, active region, phase.
- Rule variant comparison for radius, growth center, growth width, and damping.
- PNG/JSON snapshot export, share-state JSON, and evidence packs.
- Mission progress and an offline `web_html/` fallback.
- Native-first packaging through `scripts/package_submission.py`, with Windows launchers and judge placeholders.

Mission direction
-----------------
The judge should not need to tune parameters first or read a long explanation. The app should guide them through five playable tasks:

1. Wake the field by running it.
2. Shape life with brush or stamp.
3. Tune one rule and observe metric change.
4. Prove Raw Math and Artistic views are the same data.
5. Export an evidence report.

Repository rule
---------------
Keep the repository focused. Do not reintroduce secondary simulation systems unless the competition strategy changes and the teaching game remains clearer than the current Lenia-only direction.
